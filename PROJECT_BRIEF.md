# PROJECT BRIEF: CRM и База знаний репетитора

## О проекте
Внутренняя веб-платформа для одного пользователя (репетитора), автоматизирующая учёт учеников, планирование расписания и хранение учебных материалов. Доступа для учеников нет — система только для администратора.

## Стек
- **Frontend/Backend**: Next.js (App Router, TypeScript, Tailwind)
- **База данных и хранилище**: Supabase (Postgres + Storage)
- **Авторизация**: простая защита по одному паролю (один пользователь)
- **Деплой**: Vercel

## Структура (разделы сайта)
1. **Дашборд** — сводка на текущий день
2. **База учеников (CRM)** — список и карточки студентов
3. **Расписание (Календарь)** — занятость по дням/неделям/месяцам
4. **Библиотека материалов** — учебные файлы и темы

## Функциональные требования

### Дашборд
- Виджет «Сегодня»: занятия на текущий день в хронологическом порядке
- Виджет «Ближайшие задачи»: напоминания/дела по подготовке к урокам
- Быстрые действия: добавить ученика / урок / материал без перехода в раздел

### База учеников (CRM)
- Полное управление списком: добавление, удаление/архивирование
- Карточка ученика:
  - Основная информация: ФИО, контакты, уровень (Beginner, B1, Upper-Intermediate и т.д.)
  - Учебный трек: пройденные темы / темы в процессе / план на будущее
  - История и заметки: свободное текстовое поле (особенности ученика, динамика, слабые места, комментарии по урокам)

### Расписание
- Интерактивный календарь с видами «Месяц» / «Неделя» / «День»
- Управление слотами: бронирование, изменение времени, отмена в один клик
- Привязка занятия к ученику из базы

### Библиотека материалов
- Хранение пособий, статей, упражнений, методических заметок
- Добавление и удаление элементов
- Категоризация по темам/направлениям
- Фильтрация по нескольким критериям одновременно (тема, уровень сложности)
- Загрузка файлов (PDF, изображения, документы) + текстовые ссылки на внешние ресурсы

## Требования к интерфейсу
- Приоритет читаемости (scannability) — чистый интерфейс без лишних деталей
- Минимум кликов: быстрое редактирование карточек/занятий прямо из календаря и дашборда
- Адаптивность: десктоп (основное) + планшет/смартфон (для изменений «на ходу»)

## Вне рамок текущей версии (Out of Scope)
- Интеграции с внешними сервисами (Google Calendar, Notion, Zoom, онлайн-доски)
- Регистрация и личные кабинеты для учеников
- Автоматические уведомления (SMS/Email/Telegram)

> Возможные доработки на будущее (добавлять только при необходимости, отдельными итерациями):
> - Синхронизация с Google Calendar (начать с односторонней: приложение → Google Calendar)
> - Автоматическая отправка домашних заданий ученикам через персонального Telegram-бота

## План фаз разработки
1. **Фаза 1**: модель данных (Supabase-схема: students, lessons, materials, topics) + модуль «База учеников» (CRUD, карточки)
2. **Фаза 2**: модуль «Расписание» (календарь, виды месяц/неделя/день, привязка к ученику)
3. **Фаза 3**: модуль «Библиотека материалов» (загрузка файлов, категории, фильтры)
4. **Фаза 4**: Дашборд (сборка из данных предыдущих модулей)

Каждая фаза — отдельная сессия с Claude Code: реализация → проверка → коммит → деплой на Vercel.
